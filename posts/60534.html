OAuth2.0协议介绍

Flask OAuth2.0 Github
发布日期:   2020-09-05

OAuth2.0 协议

导读

个人博客https://wenyan.online 有一个评论的功能,如果用户想要评论文章,必须申请用户名和密码登录.但是,它也可以很方便的使用Github账户进行第三方登录,非常的高效.

像这样的,便捷安全的登录第三方应用的协议就是OAuth.

参考代码:https://github.com/ningwenyan/demo_code

什么是OAuth 2.0

OAuth:开放授权是一个开放标准,允许用户让第三方应用访问该用户在某一网站上存储的私密的资源(如照片,视频,联系人列表),而无需将用户名和密码提供给第三方应用.

OAuth允许用户提供一个token,而不是用户名和密码来访问他们存放在特定服务器上的数据.每一个令牌授权一个特定的网站在特定的时段内访问特定的资源.这样,OAuth让用户可以授权第三方网站访问他们存储在另外服务提供者的某些特定信息,而非所有内容.

OAuth2.0是OAuth协议的下一个版本,当不向上兼容.OAuth2.0注客户端开发者的简易性,同时为为Web应用,桌面应用,手机和智能设备提供专门的认证流程.

角色(roles)

整个协议中所有涉及到的角色

  • resource owner:资源拥有者,能够运行访问收保护资源的实体.
  • resource serve:资源服务器,托管受保护资源的服务器
  • client:客户端,第三方应用.如:web站点,移动应用等.
  • authorization server:授权服务器,即服务提供商专门用来处理认证的服务器
  • user agent:用户代理,一般为web浏览器,移动app等

协议流程

’

  • A : 客户端向资源拥有者请求授权.授权请求可以直接对资源所有者请求,也可以通过Authorization server 中转间接进行.

  • B: 资源所有者运行授权,并返回凭证

  • C:客户端通过授权服务器进行身份验证,并提供授权凭证,同时,请求token令牌

  • D:授权服务器对客户顿进行身份验证,如果有效,,则发布token令牌.

  • E:客户端通过提供token进行身份验证,进而请求资源服务器受保护的资源

  • F: 资源服务器验证访问令牌,如果正确这返回受保护资源

  • 实例: 我 使用 github账户登录wenyan.online.那么我就是resource owner ,wenyan.online这个网站就是client客户端,浏览器就是user agent,资源服务器和授权服务器都是github.

授权

根据流程可以看到,要获得access token 必须先得到授权服务器授权(authorization grant).为了得到这个授权,OAuth 2.0 定义了4种类型的授权类型

  • authorizaiton code:授权码模式,服务器与客户端配合使用.(重点)
  • implicit:隐式token,主要用于移动应用程序或web应用程序
  • resource owner password credentials :密码模式,资源所有者和客户端之间具有高度信任时,比如,客户端是操作系统的一部分才使用.
  • client credentials:客户端模式,用户直接向客户端注册,严格来说,客户端模式不属于OAuth解决的问题.

授权码模式

授权码模式是功能最网站,使用最广泛,流程最严密的授权模式.

由于这是一个基于重定向的流,所以client客户端必须能够与资源所有者的user agent(浏览器)进行交互,并且能够从授权服务器接收传入的重定向redirect请求.

  • A:用户访问client客户端,客户端采用第三方登录,并且将用户导向第三方授权服务器,通过用户代理User agent发送包括client的client identifier, requested scope, local state, and a redirection URI一并发送给授权服务器.
  • B:授权服务器对资源所有者进行身份验证,并确定资源所有者是否授予或拒绝客户端的访问请求
  • C:如果资源所有者同意授权请求,那么授权服务器将会使用指定的重定向redirection URL.跳转到客户端,并附加一个授权码code和一个本地状态码state,(如果A步骤提供,则会原值返回)
  • D:客户端收到授权码,附加上redirection URL,向授权服务器申请令牌token. 这一步是在客户端的后台服务器完成的,用户不可见.发出请求后,授权服务器对客户端进行身份验证,请求参数中包含code, client_id,client_secret, redirection URI
  • E:授权服务器对客户端进行身份验证,验证授权代码code,并确保收到的redirection URL与步骤C中的URL匹配.如果有效,授权服务器将发送access token和refresh token
  • client拿到令牌token就可以向第三方的资源服务器请求资源.

OAuth 2.0登录示例

以github为例,写一个简单的应用,演示第三方登录.

1.授权码模式登录

自建网站web,web网站让用户提供第三方网站的数据,证明自己的身份.获取第三方网站的数据,使用的是OAuth协议,按照协议,它将满足:

  • A:资源所有者user,使用client网站web在浏览器(user agent)中登录,客户端将user跳转到github.
  • B:github要求用户登录,询问web将要获取xx权限,你是否同意?
  • C:user同意后,github(授权服务器)将会重定向到web站点,同时发回一个授权码code
  • D:web站点(即客户端)使用授权码,向github请求令牌token.
  • E:Github返回令牌.
  • web站点使用令牌,向github(资源服务器)请求用户数据.

此部分实现,参照github官方文档:https://docs.github.com/cn/developers/apps/authorizing-oauth-apps

2.OAuth APP

首先,Github作为一个授权服务器,我们必须先在其服务器上登记并创建一个oauth app程序.

登录github,点击setting--Developer settings-oauth apps--new oauth app

主要参数:

  • application name:应用名称
  • homepage url:网站根目录,可以指定127.0.0.1:5000或者其他ip地址.
  • application description:描述信息
  • authorization callback url:这个选项比较重要,指定登录成功的回调地址.

提交后,Github会返回客户端ID(client id)和客户端秘钥(client secret).

3.授权oauth应用程序

名称类型描述

按照oauth协议

  • 用户:资源拥有者
  • 127.0.0.1:8080:客户端client
  • github:授权服务器
  • 浏览器:user agent
  1. 访问127.0.0.1:8080,并使用github第三方登录

  1. 点击登录跳转到github,github要求登录,并询问授予的权限,这个权限的控制来自于scope字段.

  1. 点击授权后,github重定向到之前设置的callback页面,并返回一个code授权码.到此为止,用户和github的协商就完成了,剩下的就是client和github之间的协商.

  1. 按照协议,client要使用code授权码去申请token.根据github官方说明申请如下:
POST https://github.com/login/oauth/access_token

可以衔接如下参数:

名称类型描述
client_idstringoauth app client_id必填
client_secretstringoauth app client_secret必填
codestring上一步收到的code必填
POST https://github.com/login/oauth/access_token

根据回环路由,更改oauth app的callback

  1. github返回token
  2. 使用token访问资源服务器API.对于github来说,类似如下;
Authorization: token OAUTH-TOKEN
GET https://api.github.com/user

修改代码如下:

from flask import Flask,render_template
from furl import furl
from flask import redirect
from flask import request
import requests
import json
from flask import jsonify

app = Flask(__name__)
app.config.update({
    'DEBUG':True,
    'TEMPLATES_AUTO_RELOAD' : True
})

@app.route('/')
def index():
    return  render_template('index.html')

client_id = '7f8130a3d487dbc'
client_secret = 'd359b42eaa292234fc8ed89d5bae776d'

@app.route('/redirect_github/', methods = ['GET', 'POST'])
def redirect_github():
    url = 'https://github.com/login/oauth/authorize' # 这是官网给定的地址,且必须使用 get 方法
    params = {
        'client_id': client_id, # 必填
        'scope' : 'read:user user:email'
    }
    url = furl(url).set(params)
    return redirect(url, 302)

@app.route('/oauth2/callback')
def oauth2_callback():
    # 获取code
    code = request.args.get('code')
    print(code)
    # 根据code 获取 token
    access_token_url = 'https://github.com/login/oauth/access_token'
    # 可用参数
    payload = {
        'client_id':client_id,
        'client_secret':client_secret,
        'code':code
    }
    # 发送 post
    r = requests.post(access_token_url, json=payload, headers={'Accept':'application/json'})
    print('------',json.loads(r.text))
    # 获取token
    access_toke = json.loads(r.text).get('access_token')
    print('access_token:', access_toke)

    # 获取到token后,访问资源服务器
    access_user_url = 'https://api.github.com/user'
    # 注意下方 token 后有一个空格
    rr = requests.get(access_user_url, headers={'Authorization':'token ' + access_toke})
    # 返回获取到的
    return jsonify({
        'status': 'sucess',
        'data': json.loads(rr.text)
    })


if __name__ == '__main__':
    app.run(port=8080)
